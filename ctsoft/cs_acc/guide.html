운용의 용이성

키보드 사용 보장

모든 기능은 키보드만으로도 사용가능해야 함

초점 이동

키보드에 의한 초점은 논리적으로 이동, 시각적으로 구별해야 함

응답시간 조절

시간제한이 있는 콘텐츠 응답시간 조절기능 제공

정지 기능 제공

자동으로 변경되는 콘텐츠 움직임 제어기능 제공

깜빡임과 번쩍임 사용 제한

초당 3~50회 주기로 깜빡이거나 번쩍임 금지 - 발작 일으키는 경우 있음

반복 영역 건너뛰기

반복되는 영역은 건너뛸 수 있게 스킵네비게이션(본문 바로가기) 제공

제목 제공

페이지(title), 프레임(iframe title), 콘텐츠 블록(h1, h2, h3...)

적절한 링크 텍스트

용도나 목적을 이해할 수 있도록 제공
예) 테스트를 하고 싶다면 여기를 클릭하세요 -> 이동할 페이지를 명확히 명시해야함

이해의 용이성

기본 언어 표시

주 언어 명시
예) dtd xhtml : lang="ko" xml:lang="ko" 그외 lang="ko"

사용자 요구에 따른 실행

사용자가 의도하지 않은 기능은 실행되지 않아야 함
예) 새창-팝업, 초점 변화 (onchange)

콘텐츠의 선형화

논리적인 순서로 콘텐츠 마크업 - css로 화면에 보이는 순서 변경은 상관없음 (css제거 후 html 소스로 판단)

표의 구성

표는 이해하기 쉽게 구성
- caption, summary, 헤더 셀과 데이터 셀의 관계 정의(th scope이나 id/headers)

레이블 제공

입력 서식에는 대응하는 레이블을 제공해야 함.

견고성

웹 콘텐츠는 미래의 기술로도 접근할 수 있도록 견고하게 만들어야 한다.

마크업 오류 방지

validator 체크
파이어폭스 view source나 http://validator.kldp.org에서 체크

웹 애플리케이션 접근성 준수

콘텐츠에 포함된 웹 애플리케이션(플래시, 자바스크립트, 플러그인)은 접근성을 준수해야함
예)

  • 플래시 - 플래시를 지원하지 않는 브라우저나 pc의 경우 대체컨텐츠 제공되어야 함
  • 자바스크립트 - 링크를 a태그를 이용하지 않고 div나 span 요소에 자바스크립트 클릭 이벤트를 사용하여 구현할 경우 키보드 초첨을 받지 못하거나 링크로 인식하지 못함
